# 并查集 (Disjoint Set Union)

**并查集**（DSU，又称 Union-Find）是一种用于管理元素分组关系的数据结构。它支持两种核心操作：

- **Find（查找）**：判断一个元素属于哪个集合
- **Union（合并）**：将两个集合合并为一个

## 核心思想

并查集用一个**森林**来表示集合，每个集合是一棵树，树的根节点代表该集合。初始时，每个元素自成一个集合。

### 路径压缩 (Path Compression)

在查找过程中，将沿途访问的每个节点直接指向根节点，使后续查找接近 O(1)。

```
        5                  5
       / \               / | \
      3   4     =>      3  4  2
     /                  |
    2                   2
   /
  1                    (1,2 直接连到根)
```

经过路径压缩后，所有节点几乎直接连到根，查找效率极高。

## 时间复杂度

| 操作 | 复杂度 |
|------|--------|
| Find | O(α(n)) 均摊 |
| Union | O(α(n)) 均摊 |

其中 α(n) 是反阿克曼函数，在实际中 α(n) ≤ 5，可以视为常数时间。

## 应用场景

- **最小生成树 (Kruskal)** — 判断边是否形成环
- **连通分量计数** — 统计无向图的连通块数量
- **动态连通性** — 判断两个节点是否连通
- **等价关系维护** — 如朋友圈合并问题

## 代码模板

```cpp
class DSU {
public:
    vector<int> pa;
    int sz;
    DSU(int sz) {
        this->sz = sz;
        pa.resize(sz);
        for (int i = 1; i < sz; i++) pa[i] = i;
    }
    int get(int x) {
        if (x != pa[x]) pa[x] = get(pa[x]);
        return pa[x];
    }
    void merge(int a, int b) {
        pa[get(a)] = get(b);
    }
};
```

> **提示**：上述代码使用了对 1-based 索引的初始化（`for i=1`），使用时请确保传入 `sz = n+1`。

## 可能的改进

当前的实现没有**按秩合并**（Union by Rank/Size）。加上之后可以保证树高为 O(log n)，与路径压缩结合后同样达到 O(α(n))。但对于大多数场景，仅路径压缩已经足够快。
